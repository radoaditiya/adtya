Object Oriented Programming (OOP)

OOP adalah paradigma yang berfokus pada penggunaan "object" untuk mempresentasikan data dan perilaku. Alih alih memandang program sebagai serangkaian instruksi prosedural,oop memandang sebagai kumpulan objek yang saling berinteraksi.

Konsep dasar OOP

1. objek(object):

objek adalah instance dari sebuuah class, objek memiliki state (data/atribut) dan bavior (perolaku/metode)
2.class(kelas):
class adalah blueprint atau templete untuk membuat objek,class mendefinisikan struktur dan perilaku yang akan di miliki objek objek nya

1.contoh class sederhana (car)

✖ 
      class Car {
  String brand;
  String color;
  int year;

  void start() {
    print("Mobil $brand sedang dinyalakan");
  }

  void info() {
    print("Brand: $brand");
    print("Color: $color");
    print("Year: $year");
  }
}

void main() {
  Car myCar = Car();

  myCar.brand = "Toyota";
  myCar.color = "Hitam";
  myCar.year = 2022;

  myCar.info();
  myCar.start();
}

Class adalah cetak biru (blueprint) atau rancangan untuk membuat sebuah objek. Di dalam class biasanya terdapat atribut (data) dan method (fungsi).

2.contoh constructor

✖ 
      class Produk {
  String nama;
  int harga;

  // Constructor
  Produk(this.nama, this.harga);

  void tampilkan() {
    print("Nama Produk: $nama");
    print("Harga: $harga");
  }
}

void main() {
  var produk1 = Produk("iphone", 8000000);
  var produk2 = Produk("charger", 150000);

  produk1.tampilkan();
  produk2.tampilkan();
}

Constructor digunakan untuk menginisialisasi nilai atribut saat objek dibuat.

3.contoh Validasi Data (Class Mahasiswa)

✖ 
      class Mahasiswa {
  String nama;
  String nim;
  double nilai;

  // Constructor
  Mahasiswa(this.nama, this.nim, this.nilai);

  // Method validasi
  void validasiData() {
    if (nama.isEmpty) {
      print("Nama tidak boleh kosong");
    }

    if (nim.isEmpty) {
      print("NIM tidak boleh kosong");
    }

    if (nilai < 0 || nilai > 100) {
      print("Nilai harus antara 0 - 100");
    }
  }

  void tampilkanData() {
    print("Nama: $nama");
    print("NIM: $nim");
    print("Nilai: $nilai");
  }
}

void main() {
  Mahasiswa mhs1 = Mahasiswa("Rado Aditiya", "251410012", 100);

  mhs1.validasiData();
  mhs1.tampilkanData();
}

Contoh ini menunjukkan validasi data pada constructor untuk memastikan IPK dalam rentang 0.0 - 4.0.

4.contoh Encapsulation

✖ 
      class RekeningBank {
  String _namaPemilik;
  double _saldo;

  // Constructor
  RekeningBank(this._namaPemilik, this._saldo);

  // Getter
  double getSaldo() {
    return _saldo;
  }

  // Setter
  void setSaldo(double jumlah) {
    _saldo = jumlah;
  }

  void tampilkanData() {
    print("Nama Pemilik: $_namaPemilik");
    print("Saldo: $_saldo");
  }
}

void main() {
  var rekening = RekeningBank("Rado", 1000000);

  rekening.tampilkanData();

  // Mengubah saldo menggunakan setter
  rekening.setSaldo(2000000);

  print("Saldo baru: ${rekening.getSaldo()}");
}

Encapsulation membatasi akses langsung ke atribut menggunakan setter dan getter.

5.contoh Constructor dengan Named Parameter

✖ 
      class Produk {
  String nama;
  int harga;
  String kategori;

  // Constructor dengan Named Parameter
  Produk({required this.nama, required this.harga, required this.kategori});

  void tampilkanProduk() {
    print("Nama Produk: $nama");
    print("Harga: $harga");
    print("Kategori: $kategori");
  }
}

void main() {
  var produk1 = Produk(
    nama: "Laptop",
    harga: 8000000,
    kategori: "Elektronik"
  );

  produk1.tampilkanProduk();
}

Named parameter constructor membuat kode lebih rapi dan mudah dibaca.
